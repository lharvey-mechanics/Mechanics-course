Key terms

Sensor

Device that measures a condition and sends a signal.

In the shop: Coolant temperature sensor reporting to the PCM.

Actuator

Device controlled by a module to perform an action.

In the shop: PCM commanding a cooling fan relay on.

Input signal

Information a module receives from sensors.

In the shop: TPS voltage changing as throttle opens.

Output command

A module’s control signal to an actuator.

In the shop: Commanding a relay or solenoid on/off.

Reference voltage

A stable voltage (often 5 V) provided to sensors.

In the shop: Testing for 5 V reference at a MAP sensor.

Signal return

The sensor’s output back to the module.

In the shop: Measuring sensor signal voltage at the connector.

CAN bus

Vehicle network used for module communication.

In the shop: Multiple warning lights due to CAN communication fault.

Scan tool data

Live readings and information from vehicle modules.

In the shop: Checking fuel trim values during diagnosis.

PID

A named scan data item (parameter ID).

In the shop: Viewing engine rpm PID vs tachometer.

DTC

Diagnostic Trouble Code indicating a fault.

In the shop: P0101 related to MAF sensor performance.

Freeze frame

Snapshot of operating conditions when a code set.

In the shop: Using freeze frame to see speed/load at failure.

Drive cycle

Specific conditions needed to run OBD monitors.

In the shop: Completing a drive cycle to set readiness.
